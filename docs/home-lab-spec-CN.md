# 实施规范：2.5 家庭实验室隐私优先原型 (Implementation Specification: 2.5 Home-Lab Privacy-First Prototype)

## 1. 概述
家庭实验室是微型托儿所的前身，旨在受控的家庭环境中验证“感官采集流水线”。目标是在不进行侵入式监视的情况下，映射饮食偏好和感官反应。

## 2. 硬件架构
- **视觉**: 
    - **设备**: USB 摄像头 (例如 Logitech C920 或针孔摄像头)。
    - **布置**: 隐藏在天堂鸟植物中，定位为直接的桌面俯视视角。
    - **约束**: **隐私设计 (Privacy-by-Design)**。仅采集桌面区域；不记录面部或身体。
- **音频**: 
    - **设备**: USB 麦克风。
    - **采集**: 专注于用餐区域的高保真音频。
- **身份识别**: 
    - **标记**: 由孩子佩戴的物理颜色代码标记 (例如腕带)。
    - **检测**: 基于 OpenCV 的颜色过滤，以识别当前在桌边的孩子。
- **计算**:
    - **主机**: Mac/Linux 机器。
    - **角色**: 数据摄取和异步处理。

## 3. 软件流水线 (异步架构)

### 阶段 A: 实时采集 (采集器 - The Collector)
- **视频流**: 定期帧采集 ➡️ 保存为带有时间戳的 `.jpg` 文件。
- **音频流**: 持续录音 ➡️ 分段为 `.wav` 块。
- **身份跟踪**: 持续监测标记颜色 ➡️ 记录身份变化。

### 阶段 B: 异步分析 (分析器 - The Analyzer)
- **视觉处理**: 
    - 食物检测与分类。
    - 在身份触发期间对存在的食物进行上下文映射。
- **音频处理**: 
    - 针对关键词的语音转文本 (STT)：“Oui” (是)、“Non” (否) 以及其他偏好指标。
    - 将音频响应与食物图像进行时间对齐。
- **综合分析**:
    - 将 `[身份] + [食物图像] + [音频响应]` 聚合为结构化的“感官日志”。

## 4. 核心哲学与约束
- **隐私优先**: 不存储或处理面部数据。
- **以主体为中心**: 将孩子的*选择* (Oui/Non) 作为自主能力的标志。
- **低摩擦**: 使用现有的家庭基础设施 (植物、改造的麦克风) 以最大限度地减少环境压力。
